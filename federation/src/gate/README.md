# The Gate - Public WebSocket Gateway\n\nInternet-facing entry point for Manifold mesh access. Agents show their MeshPass, walk through The Gate, and they're on the mesh. No Tailscale needed.\n\n## Overview\n\n**The Gate** is a WebSocket server that provides public access to the Manifold mesh federation. It handles:\n\n- **Authentication** - Verify MeshPass credentials via Ed25519 signatures\n- **Authorization** - Rate limiting and session management\n- **Bridge** - Connect public clients to private federation servers\n- **Security** - IP-based limits, message validation, timeout enforcement\n\n## Quick Start\n\n### 1. Start The Gate Server\n\n```typescript\nimport { Gate } from './index.js'\n\nconst gate = new Gate({\n  port: 8765,                              // Public WebSocket port\n  hubName: 'satelliteA',                   // This hub's name\n  federationServer: 'ws://localhost:8766', // Internal federation server\n  maxConnectionsPerIP: 10,                 // Rate limiting\n  maxMessagesPerSecond: 50,\n  debug: true\n})\n\n// Register known MeshIDs (in production, use a proper registry)\ngate.registerMeshID('stella@satelliteA', 'abc123...publickey...789')\ngate.registerMeshID('eddie@satelliteA', 'def456...publickey...012')\n\nawait gate.start()\nconsole.log('The Gate is open on port 8765!')\n```\n\n### 2. Connect Client to The Gate\n\n```typescript\nimport { GateClient } from './client.js'\nimport { MeshPass } from '../identity/index.js'\n\n// Load your MeshPass\nconst meshPass = await MeshPass.load('your-passphrase')\n\n// Connect to The Gate\nconst client = new GateClient({\n  gateUrl: 'wss://gate.satelliteA.org:8765',\n  meshPass,\n  meshId: 'stella@satelliteA',\n  debug: true\n})\n\n// Set up event handlers\nclient.on('authenticated', (session) => {\n  console.log('Authenticated!', session)\n})\n\nclient.on('message', (message) => {\n  console.log('Received:', message)\n})\n\nawait client.connect()\n// Authentication happens automatically after connection\n\n// Send messages\nclient.send({\n  type: 'capability_query',\n  capability: 'solar-monitoring',\n  requestId: 'query-123'\n})\n```\n\n## Architecture\n\n```\n[Internet] → [The Gate] → [Federation Server] → [Tailscale Mesh]\n           ↑           ↑                     ↑\n        WebSocket    Bridge              Internal mesh\n      Authentication Connection          (existing)\n```\n\n**The Gate** sits between the public internet and your internal federation infrastructure:\n\n1. **Public Layer**: WebSocket server on public internet\n2. **Auth Layer**: MeshPass credential verification\n3. **Bridge Layer**: Forward authenticated messages to federation server\n4. **Federation Layer**: Existing Tailscale-based mesh (unchanged)\n\n## Authentication Flow\n\n```\nClient                    The Gate                Federation Server\n  │                         │                           │\n  ├─── WebSocket Connect ───→│                           │\n  │←── gate_info ────────────│                           │\n  │                         │                           │\n  ├─── mesh_auth ───────────→│                           │\n  │    (signed message)       │                           │\n  │                         │                           │\n  │←── auth_success ─────────│                           │\n  │    (session established)  │                           │\n  │                         │                           │\n  ├─── federation_message ──→│─── enriched_message ────→│\n  │                         │    (+ sender identity)    │\n  │                         │                           │\n  │←── federation_response ──│←── response ─────────────│\n```\n\n1. **Connect**: Client opens WebSocket to The Gate\n2. **Info**: Gate sends welcome message with hub info\n3. **Auth**: Client sends signed authentication message\n4. **Verify**: Gate verifies signature against MeshID registry\n5. **Session**: Gate creates authenticated session\n6. **Relay**: Gate relays messages between client and federation\n\n## Configuration\n\n```typescript\ninterface GateConfig {\n  /** Port to listen on for public WebSocket connections */\n  port: number\n  \n  /** Hub name this gate serves */\n  hubName: string\n  \n  /** Federation server address to bridge to */\n  federationServer: string\n  \n  /** Rate limiting: max connections per IP (default: 10) */\n  maxConnectionsPerIP?: number\n  \n  /** Rate limiting: max messages per second per connection (default: 50) */\n  maxMessagesPerSecond?: number\n  \n  /** Session timeout in milliseconds (default: 30 minutes) */\n  sessionTimeoutMs?: number\n  \n  /** Authentication timeout in milliseconds (default: 30 seconds) */\n  authTimeoutMs?: number\n  \n  /** Enable debug logging */\n  debug?: boolean\n}\n```\n\n## Rate Limiting\n\nThe Gate implements multiple layers of rate limiting:\n\n### Connection Limits\n- **Per-IP**: Max concurrent connections from same IP\n- **Auth timeout**: Max time to complete authentication\n- **Session timeout**: Max idle time before session expires\n\n### Message Limits\n- **Per-second**: Max messages per second per connection\n- **Burst protection**: Reset counters every second\n- **Auth attempts**: Max failed auth attempts before disconnect\n\n### Error Handling\n- **Graceful degradation**: Rate limits return errors, don't crash\n- **Automatic cleanup**: Expired sessions cleaned up periodically\n- **Connection tracking**: IP-based connection counting\n\n## Security Features\n\n### Cryptographic Authentication\n- **Ed25519 signatures** - Industry standard elliptic curve cryptography\n- **Timestamp validation** - Prevents replay attacks (5 minute window)\n- **Nonce verification** - Each auth request has unique nonce\n- **Public key verification** - Messages verified against registered keys\n\n### Network Security\n- **IP rate limiting** - Prevents DoS attacks\n- **Connection limits** - Per-IP connection caps\n- **Message validation** - All messages parsed and validated\n- **Timeout enforcement** - No hanging connections\n\n### Session Management\n- **Authenticated sessions** - Only verified MeshPass holders get through\n- **Session tracking** - Monitor active sessions and activity\n- **Automatic cleanup** - Expired sessions removed automatically\n- **Identity enrichment** - Messages tagged with verified sender identity\n\n## Monitoring\n\n```typescript\n// Get gate statistics\nconst stats = ({ meshId, publicKey }) => {\n  console.log(`${meshId} authenticated`)\n})\n\ngate.on('session:expired', ({ meshId }) => {\n  console.log(`${meshId} session expired`)\n})\n```\n\n### Gate Client Events\n```typescript\nclient.on('connected', () => {\n  console.log('Connected to The Gate')\n})\n\nclient.on('authenticated', (session) => {\n  console.log('Authenticated:', session)\n})\n\nclient.on('auth_error', (error) => {\n  console.error('Auth failed:', error)\n})\n\nclient.on('message', (message) => {\n  // Handle federation messages\n})\n```\n\n## Error Codes\n\n- `4000` - Authentication timeout\n- `4001` - Too many authentication failures\n- `RATE_LIMIT` - Message rate limit exceeded\n- `FEDERATION_DOWN` - Federation server unavailable\n- `INVALID_FORMAT` - Message parsing failed\n- `PROCESSING_ERROR` - Message processing failed\n\n## Production Deployment\n\n### SSL/TLS\nAlways use WSS (WebSocket Secure) in production:\n\n```javascript\n// Use a reverse proxy like nginx for SSL termination\n// The Gate runs on localhost, nginx handles public SSL\nconst gate = new Gate({\n  port: 8765,        // localhost only\n  // ... nginx forwards from public :443 to localhost:8765\n})\n```\n\n### Process Management\n```javascript\n// Graceful shutdown\nprocess.on('SIGTERM', async () => {\n  console.log('Shutting down The Gate...')\n  await gate.stop()\n  process.exit(0)\n})\n```\n\n### Monitoring\n```javascript\n// Health check endpoint\nsetInterval(() => {\n  const stats = gate.getStats()\n  if (stats.sessions.authenticated > 1000) {\n    console.warn('High session count:', stats.sessions.authenticated)\n  }\n}, 60000)\n```\n\n### Registry Management\nIn production, use a proper identity registry:\n\n```javascript\n// Load MeshIDs from database/config\nconst registry = await loadMeshIDRegistry()\nfor (const [meshId, publicKey] of registry) {\n  gate.registerMeshID(meshId, publicKey)\n}\n```\n\n## Troubleshooting\n\n### Authentication Failures\n1. **Check MeshID format** - Must be `name@hub`\n2. **Verify public key** - Must be registered with The Gate\n3. **Check signature** - Message must be properly signed\n4. **Timestamp validation** - Messages expire after 5 minutes\n\n### Connection Issues\n1. **Rate limiting** - Check if IP has too many connections\n2. **Federation server** - Ensure gate can reach federation server\n3. **Network** - Verify WebSocket connectivity\n4. **SSL/TLS** - Check certificate validity for WSS\n\n### Debug Mode\nEnable debug logging:\n\n```javascript\nconst gate = new Gate({ ...config, debug: true })\nconst client = new GateClient({ ...config, debug: true })\n```\n\n## Examples\n\nSee `federation/examples/` for complete examples:\n- Basic gate setup\n- Client
