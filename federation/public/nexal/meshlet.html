? What is a Meshlet?

A meshlet is an isolated workspace for prototyping new agents before they join the live mesh. Think of it as a sandbox — you can experiment without affecting anything real.

HOW IT WORKS
1. Launch
Creates a fresh meshlet instance with its own memory, void space, and agent registry. Runs for 2 hours.
2. Open Voids
Voids are exploration spaces for concepts. Enter a term like agent-identity or deployment-strategy. Related concepts create pressure — when pressure is high enough, the void can be "graduated" into a named agent.
3. Register Agents
Use the Agent Builder to define an agent with specific capabilities and seams. Or name a void to graduate it automatically.
4. Scan Signals
SSJ2 reach scan detects seams — overlapping capability gaps between agents. These show where new agents could add value to the mesh.
EXAMPLE WORKFLOWS
Explore a concept:
Launch → Open void solar-prediction → Add implied-by terms flare-detection, cme-tracking → Nudge pressure up → Name it → solar-predictor agent is born
Build an agent directly:
Launch → Skip voids → Go to Agent Builder → Name it, pick capabilities, define seams → Register → Run SSJ2 scan to see where it fits in the mesh
Find mesh gaps:
Launch → Run SSJ2 reach scan → See which seams have high signal strength → Open voids for those areas → Graduate into agents that fill the gaps
