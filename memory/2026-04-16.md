# 2026-04-16 Session Memory

## HOG OOM Crash — Root Cause & Fix

**Problem:** HOG federation node kept crashing with OOM. Two separate broadcast storms found and fixed.

**Storm 1 (claims):** `DetectionLedger.addClaim()` returned truthy entry for duplicates. `DetectionCoord.handleClaim()` broadcast every claim to all peers, including duplicates. Two hubs ping-ponged claims infinitely.
- Fix: `addClaim()` returns `null` for duplicates. `handleClaim()` skips broadcast on null.

**Storm 2 (verifications — the real killer):** `auto-verify@satelitea` re-verified open claims every 15 min (verification doesn't close claims, only outcomes do). Each verification broadcast to HOG → HOG re-broadcast back → satelitea re-broadcast → loop. **1.35 million verification messages for a single claim in 2 hours.**
- Fix: `addVerification()` and `addChallenge()` return `null` for duplicates. `handleVerify()` and `handleChallenge()` skip broadcast on null.

**Result:** Both nodes stable at ~96-102MB. No memory growth.

- `federation/src/server/detection-ledger.ts` — dedup in addClaim, addVerification, addChallenge
- `federation/src/server/detection-coord.ts` — null-check before broadcast in handleClaim, handleVerify, handleChallenge
